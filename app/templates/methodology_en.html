How We Measure Password Strength

Methodology behind the strength score and crack time

1. Entropy-based estimation

OxePass generates passwords using Python's secrets module — a cryptographically secure random number generator. Because we control the generation and can guarantee true randomness, we use information entropy to measure strength.

Entropy quantifies how unpredictable a password is. The formula is:

E = L × log₂(P)

Where L = password length and P = size of the character pool

2. Character pool

The pool size depends on which character types are present in the password:

Lowercase (a–z) 26
Uppercase (A–Z) 26
Digits (0–9) 10
Symbols (!@#$%...) 32

For example, a password with lowercase + uppercase + digits + symbols has a pool of 94 characters. A 16-character password with this pool has 104.8 bits of entropy.

3. Attack scenario

To calculate crack time, we simulate the most aggressive realistic scenario:

Offline fast hashing — 10 billion attempts/second

Simulates a cluster of modern GPUs performing a brute force attack against stolen hashes using algorithms like MD5 or SHA-1 (without salting). This is the worst-case scenario for the user.

The estimated time represents how long an attacker with this equipment would take, on average, to find the password — that is, 50% of the total search space:

T = 2E ÷ (2 × 10¹⁰)

Where E = entropy in bits and 10¹⁰ = guesses per second

4. Strength score

The score is derived directly from the crack time, so they are always consistent:

Very Weak
< 10 minutes
Weak
< 1 day
Moderate
< 1 year
Strong
< 1,000 years
Very Strong
≥ 1,000 years

5. Why entropy works for generated passwords

There are two different scenarios for measuring password strength:

Human-chosen passwords

People tend to use dictionary words, dates, keyboard patterns and predictable substitutions. Tools like zxcvbn are designed for this case, decomposing the password into the cheapest combination of known patterns.

Machine-generated passwords

When generated with a cryptographic RNG like Python's secrets module, every character is independently and uniformly random. There are no patterns to exploit, so entropy is the exact measure of difficulty.

Since OxePass generates all passwords with secrets, an attacker's only option is brute force over the full character pool. This is exactly what the entropy formula calculates.

6. Practical example

A password from the Personal profile (16 characters, all types):

Pool: 26 + 26 + 10 + 32 = 94

Entropy: 16 × log₂(94) = 104.8 bits

Search space: 2¹⁰⁴·⁸ ≈ 2.04 × 10³¹ combinations

Average time: ~1.02 × 10²¹ seconds ≈ 32 trillion years

7. Important caveats

Crack time is an estimate

Real time depends on the hashing algorithm used by the service. For example, bcrypt is much slower than MD5, reducing the attack rate to thousands per second.

Valid only for randomly generated passwords

If you type your own password, patterns and dictionary words could drastically reduce the real entropy.

A strong password does not replace 2FA

Enable two-factor authentication whenever possible. Even the strongest password can be compromised by phishing or a data breach.
